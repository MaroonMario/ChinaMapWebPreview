> 版本：v1.0  
> 状态：v1 P0 已完成，详见 [开发进度](./PROGRESS.md)  
> 最后更新：2026-06-28

---

## Problem Statement

很多人需要熟悉中国行政区划（省/自治区/直辖市/特别行政区），但纸质地图或静态图片缺乏反馈，难以建立「名称 ↔ 位置 ↔ 形状」的记忆关联。

**目标用户**：自学者、学生、备考者，或需要快速查阅省界与省会的人。

**问题频率**：备考或旅行前集中使用；日常偶尔查阅。

**不解决的代价**：学习效率低、容易混淆相邻省份，也难以通过自测验证掌握程度。

---

## Goals

| # | 目标 | 衡量方式 |
|---|------|----------|
| G1 | 用户能在 30 秒内找到任意指定省份 | 探索模式下计时任务 |
| G2 | 用户能通过交互建立省名与位置的对应关系 | 测验模式正确率 ≥ 80%（连续 10 题） |
| G3 | 核心流程无需注册、打开即用 | 首屏 3 秒内可交互（本地/静态部署） |
| G4 | 移动端可用（平板/手机竖屏可读） | 375px 宽度下地图可缩放、信息可读 |
| G5 | 为 v2 扩展（市级、专题图层）预留数据与架构空间 | 地图数据与 UI 逻辑解耦 |

---

## Non-Goals（v1 明确不做）

1. **反向测验（认名称）** — v1 只做「认位置」，从 4 选项中选省名留 v2
2. **精确到市/县级 drill-down** — 数据量大、交互复杂，留 v2
3. **用户账号、进度云同步、排行榜** — 个人学习工具，本地存储即可
4. **实时地理信息（交通、天气、人口统计）** — 与「熟悉地图」核心目标无关
5. **3D 地球 / 复杂 GIS 分析** — 2D 矢量地图足够
6. **多语言（英文省名等）** — v1 以中文为主，架构上不阻塞后续 i18n

## User Stories

### 探索者（自由浏览）

- 作为**地理自学者**，我希望**鼠标悬停/触摸省份时看到名称**，以便**建立视觉记忆**。
- 作为**自学者**，我希望**点击省份后在侧边面板看到详细信息**（全称、简称、省会、所属区域），以便**加深理解**。
- 作为**移动端用户**，我希望**双指缩放、拖拽地图**，以便**在小屏幕上查看细节**。

### 测验者（认位置练习）

- 作为**备考者**，我希望**系统随机出题「请点击：XX 省」**，以便**检验是否认得出位置**。
- 作为**备考者**，我希望**答对/答错有即时反馈并显示正确答案**，以便**纠正错误记忆**。
- 作为**备考者**，我希望**查看本轮正确率**，以便**了解掌握程度**。

### 边界与异常

- 作为**用户**，当**点击南海诸岛等较小区域**时，我希望**仍能选中并看到名称**，以便**版图完整**。
- 作为**用户**，当**网络较慢加载 GeoJSON** 时，我希望**看到加载状态**，而不是空白页。

## Requirements

### Must-Have（P0）

| ID | 需求 | 验收标准 | 状态 |
|----|------|----------|------|
| R1 | 省级矢量中国地图 | 34 个省级单元（含港澳台）边界正确渲染；默认居中、适配视口 | ✅ |
| R2 | 悬停高亮 | Given 地图已加载，When 悬停某省，Then 该省高亮且显示省名 tooltip | ✅ |
| R3 | 点击详情 | Given 地图已加载，When 点击某省，Then 面板展示：全称、简称、省会、所属大区 | ✅ |
| R4 | 缩放与平移 | 支持滚轮/双指缩放、拖拽平移；提供「重置视图」按钮 | ✅ |
| R5 | 测验模式：认位置 | Given 进入测验，When 题目为「请点击：四川省」，Then 点击正确省得分+1，错误显示正确位置并进入下一题 | ✅ |
| R6 | 模式切换 | 探索 / 测验 两种模式可切换，状态互不干扰 | ✅ |
| R7 | 响应式布局 | 375px–1920px 下地图与信息面板可用，无横向溢出 | ✅ |
| R8 | 静态部署 | `npm run build` 后可部署到任意静态托管 | ✅ |

### Nice-to-Have（P1）

| ID | 需求 | 验收标准 | 状态 |
|----|------|----------|------|

| R10 | 本地进度 | `localStorage` 保存最近测验成绩与错题省列表 | ⬜ |

## 技术方案

| 层级 | 选型 | 理由 |
|------|------|------|
| 框架 | React + Vite + TypeScript | 生态成熟、类型安全 |
| 地图渲染 | ECharts `map` + GeoJSON | 中国省界资源现成、交互完善 |
| 数据 | 静态 JSON：`public/china.json` + `src/data/provinces.ts` | 无后端、易维护 |
| 样式 | 纯 CSS | 轻量、符合白底蓝高亮需求 |
| 状态 | React Hooks | 模式、选中省、测验状态 |

**GeoJSON 来源**：[阿里云 DataV 省级边界](https://geo.datav.aliyun.com/areas_v3/bound/100000_full.json)

**省份元数据字段**：

```json
{
  "adcode": 510000,
  "name": "四川省",
  "shortName": "川",
  "capital": "成都市",
  "region": "西南"
}
```

## Success Metrics

### Leading（上线后 1–2 周）

- 探索模式：任意指定 5 省，平均定位时间 < 15 秒
- 测验模式：10 题一轮，完成率 100%（无崩溃/卡死）
- Lighthouse Performance ≥ 85（生产构建）

### Lagging（长期使用）

- 同一用户多次测验后，认位置正确率从 < 50% 提升到 ≥ 80%
- 每周至少打开 3 次（若做本地统计）
